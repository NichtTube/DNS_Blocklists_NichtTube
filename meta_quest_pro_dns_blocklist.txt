! Title: NichtTube's Meta Quest Pro Update Blocklist
! Description: Blocks Meta Quest Pro OTA update check endpoints. Goal: prevent automatic firmware updates without breaking login, store, pairing, or social features - and without collateral damage on other devices when applied network-wide.
! Version: 1.1.0
! Homepage: https://github.com/NichtTube/DNS_Blocklists_NichtTube
! Last modified: 29.04.2026
! Syntax: AdGuard
!
! USAGE NOTE
! When applied via AdGuard Home or AdGuard DNS at network level, assign this
! list to the Quest device ONLY (per-client rule / per-profile assignment).
! Several endpoints are shared with other Meta services; network-wide
! application beyond the Quest is not the intended scope of this list.
!
# === OTA update check endpoints (Quest / Oculus) ===
# Primary update check path. Reverse-engineered from the Quest 2 updater,
# see https://github.com/Migushthe2nd/QuestFirmwareGrabber.
# Blocking these is sufficient to suppress automatic firmware updates -
# without these, the device never receives the dynamic CDN download URL.
||graph.oculus.com^
||graph-fallback.oculus.com^
||realitylabs-graph.oculus.com^
||graph.facebook-hardware.com^

# === Oculus CDN (firmware payload delivery) ===
# *.oculuscdn.com is Oculus-only -> safe to widen vs. a regional pin.
||oculuscdn.com^
||securecdn.oculus.com^

# === Quest Pro device endpoints ===
||horizon.meta.com^
||portal.fb.com^
||b-www.facebook.com^

# === OPTIONAL: aggressive extras ===
# Disabled by default - these hosts are shared with Facebook, Instagram,
# Messenger, WhatsApp and any third-party app that uses the Facebook SDK.
# Enable individually only if this list is scoped to the Quest device.
! ||graph.facebook.com^
! ||edge-mqtt.facebook.com^
! ||gateway.facebook.com^
