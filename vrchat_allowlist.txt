!
! Title: NichtTube's VRChat Allowlist
! Description: Minimal allowlist for VRChat core infrastructure. Designed to NOT interfere with ad/tracker blocking on shared media hosts (YouTube, Twitch, Imgur, etc.). Only domains that are exclusively VRChat or Photon are exempted.
! Version: 1.1.1
! Homepage: https://github.com/NichtTube/DNS_Blocklists_NichtTube
! Last modified: 30.04.2026
! Syntax: AdGuard
!
! USAGE NOTE
! This list deliberately does NOT allowlist YouTube, Twitch, Imgur, Twitter
! or other shared media hosts. Reason: $important would override your ad
! and tracking blocks on those services. DNS-level blocklists do not block
! these domains at the apex anyway - their ad blocking happens elsewhere.
!
! If a specific VRChat world fails to load video/image content, add a
! TARGETED, SCOPED rule for that exact subdomain via your AdGuard Home
! per-client config - do NOT widen this list.
!
# === Core VRChat infrastructure (REQUIRED) ===
@@||vrchat.com^$important
@@||vrchat.cloud^$important

# === Photon realtime networking (REQUIRED) ===
# Without these: ClientTimeout on every instance join.
# These domains are Photon-exclusive - no ad/tracker collateral.
@@||exitgames.com^$important
@@||photonengine.io^$important

# Specific distribution IDs - not the wildcard *.cloudfront.net.
# Sourced from api.vrchat.cloud/api/1/config (whiteListedAssetUrls + urlList).

@@||dqrpb9wgowsf5.cloudfront.net^$important
@@||ds0h3roq6wcgc.cloudfront.net^$important
@@||d2nvs31859zcd8.cloudfront.net^$important

# === Unity services (REQUIRED) ===
# Game engine runtime services. Not in any major DNS blocklist.
@@||unity3d.com^$important
@@||unity.com^$important

# === DELIBERATELY NOT INCLUDED ===
# The following were considered and rejected to preserve your ad blocking:
#   youtube.com, googlevideo.com, ytimg.com - YT ad-blocking lives in
#     browser extension; DNS allowlist would not affect that anyway, but
#     would defeat any tracking subdomain blocks.
#   twitch.tv, ttvnw.net, twitchcdn.net, jtvnw.net - same logic.
#   imgur.com, twimg.com, redd.it - tracking pixels on these ARE typically
#     blocked at DNS level. Allowlisting them defeats privacy protection.
#   facebook.com, fbcdn.net - conflicts with meta_quest_pro_dns_blocklist.
#
# If you encounter a VRChat world that needs one of these, add the SPECIFIC
# subdomain (e.g. @@||i.imgur.com^$important) to AdGuard Home per-client
# rules - not to this network-wide list.
